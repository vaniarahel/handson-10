## Catatan Pembelajaran
1. SRP (Single Responsibility Principle): Setiap kelas harus memiliki satu dan hanya satu alasan untuk diubah, memastikan bahwa perubahan pada satu fungsionalitas tidak merusak fungsionalitas lain.
2. OCP (Open/Closed Principle): Kode harus terbuka untuk diperluas (penambahan fitur baru) tetapi tertutup untuk dimodifikasi, yang dicapai dengan mengandalkan abstraksi (interface atau abstract class).
3. LSP (Liskov Substitution Principle): Objek dari superclass harus dapat digantikan dengan objek dari subclass tanpa mengubah perilaku yang diharapkan atau merusak keandalan program (polymorphism).
